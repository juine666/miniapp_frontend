# uni-app 开发模式说明

## 开发时的自动编译

### 方法1：使用 HBuilderX（推荐，最简单）

**✅ HBuilderX 有自动编译功能！**

1. **打开项目后，点击"运行"按钮**（绿色▶️）
2. **选择"运行到小程序模拟器 → 微信开发者工具"**
3. **之后每次修改代码：**
   - 保存文件（Cmd+S 或 Ctrl+S）
   - HBuilderX 会自动检测文件变化
   - 自动重新编译
   - 微信开发者工具会自动刷新

**不需要手动点击编译！** 只要保持 HBuilderX 运行状态，修改代码保存后就会自动编译并刷新。

### 方法2：使用命令行 Watch 模式

如果你更喜欢用命令行或编辑器：

```bash
# 进入项目目录
cd miniapp-frontend-uniapp

# 开启 watch 模式（自动监听文件变化）
npm run dev:mp-weixin
```

**特点：**
- 启动后会自动监听文件变化
- 修改代码保存后，自动重新编译
- 需要在微信开发者工具中手动刷新（F5）

### 方法3：继续使用微信小程序原生代码（最简单）

如果你觉得编译麻烦，可以继续使用原来的 `miniapp-frontend` 项目：

**优点：**
- ✅ 不需要编译，直接修改代码
- ✅ 保存后立即生效（微信开发者工具自动刷新）
- ✅ 更快的开发体验

**缺点：**
- ❌ 不能跨平台（只能用于微信小程序）
- ❌ 代码是原生小程序格式（WXML、WXSS、JS）

## 推荐工作流程

### 如果使用 HBuilderX：

1. **第一次运行**：
   - 打开 HBuilderX
   - 导入项目
   - 点击运行 → 运行到小程序模拟器 → 微信开发者工具

2. **日常开发**：
   - 保持 HBuilderX 和微信开发者工具都打开
   - 在 HBuilderX 中修改代码
   - 保存文件（Cmd+S）
   - **自动编译并刷新**，无需手动操作

### 如果使用命令行 + VS Code/Cursor：

1. **第一次启动**：
   ```bash
   cd miniapp-frontend-uniapp
   npm run dev:mp-weixin
   ```
   然后在微信开发者工具中导入 `dist/dev/mp-weixin` 目录

2. **日常开发**：
   - 保持命令行窗口运行（watch 模式）
   - 在编辑器（VS Code/Cursor）中修改代码
   - 保存文件
   - 自动编译，然后在微信开发者工具中按 F5 刷新

## 性能对比

| 方式 | 编译速度 | 刷新速度 | 开发体验 |
|------|---------|---------|---------|
| HBuilderX 自动编译 | ⚡⚡⚡ 很快 | ⚡⚡⚡ 自动刷新 | ⭐⭐⭐⭐⭐ 最好 |
| 命令行 Watch | ⚡⚡ 快 | ⚡ 需手动刷新 | ⭐⭐⭐⭐ 好 |
| 微信小程序原生 | ⚡⚡⚡⚡ 无需编译 | ⚡⚡⚡ 自动刷新 | ⭐⭐⭐⭐⭐ 最快 |

## 建议

**如果你主要在微信小程序平台开发：**
- ✅ **推荐继续使用原来的 `miniapp-frontend` 项目**
- 无需编译，修改代码立即生效
- 开发体验最流畅

**如果你需要跨平台（微信、支付宝、H5等）：**
- ✅ **使用 uni-app + HBuilderX**
- 修改代码保存后自动编译
- 一次编写，多端运行

## 总结

**不需要每次手动编译！**

- **HBuilderX**：保存文件后自动编译并刷新
- **命令行 Watch**：保存文件后自动编译，手动刷新
- **微信小程序原生**：无需编译，直接生效

选择最适合你的方式即可！
